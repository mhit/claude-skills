## 推奨モデル

**`gemini-3-flash-preview`** - バランス型、速度と品質の両立

## いつ使うか

- 新しいUIコンポーネントを設計したい
- ユーザーフローを最適化したい
- デザインシステムを構築したい
- 既存UIの改善提案がほしい
- モックアップやワイヤーフレームのアイデアがほしい

## Geminiの強み

- UI/UXパターンの豊富な知識
- ユーザビリティのベストプラクティス
- アクセシビリティへの配慮
- モダンなデザイントレンドの把握

## 実行手順

### 1. 要件の整理

設計対象について以下を明確にする:
- 目的・ゴール
- ターゲットユーザー
- 既存のデザインシステム（あれば）
- 技術的制約

### 2. Gemini呼び出し

```bash
# UI設計の相談（Gemini 3 Flash推奨 - バランス型）
gemini -m gemini-3-flash-preview "以下の要件に基づいてUIを設計してください: [要件]"

# 既存UIの改善
gemini -m gemini-3-flash-preview "以下のUIを分析し、改善提案をしてください" < /tmp/current_ui_description.txt

# gemini-call.shを使う場合
gemini-call.sh --task design "ダッシュボードのUI設計を提案して"
```

### 3. プロンプト例

**新規UI設計**:
```
以下の要件でUIを設計してください:

目的: [例: ユーザー登録フォーム]
ターゲット: [例: 初めてのユーザー]
デバイス: [例: モバイルファースト]
制約: [例: Tailwind CSS使用]

出力形式:
1. ワイヤーフレーム（テキスト表現）
2. コンポーネント構成
3. インタラクションフロー
4. アクセシビリティ考慮事項
```

**UI改善提案**:
```
以下のUIを分析し、改善提案をしてください:

[現在のUI説明またはスクリーンショット]

観点:
1. ユーザビリティ
2. 視覚的階層
3. アクセシビリティ
4. レスポンシブ対応
5. マイクロインタラクション
```

**デザインシステム構築**:
```
以下のプロダクトのデザインシステムを設計してください:

プロダクト概要: [説明]
ブランドカラー: [あれば]
既存コンポーネント: [あれば]

出力:
1. カラーパレット
2. タイポグラフィ
3. スペーシングシステム
4. 基本コンポーネント一覧
5. コンポーネント階層
```

## 結果の活用

Geminiの設計提案をClaudeが実装:

1. 提案されたUIをReact/HTML/CSSで実装
2. デザイントークンをコード化
3. コンポーネントライブラリの構築
4. アクセシビリティテストの実施

## Tips

- 具体的な要件を伝えるほど、具体的な提案が得られる
- スクリーンショットを含めると、より的確なフィードバックが得られる
- 複数の選択肢を求めて比較検討する
- 実装フェーズではClaudeに引き継ぐ
