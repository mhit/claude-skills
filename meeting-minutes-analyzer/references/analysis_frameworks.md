## 意思決定ツリー分析

### 目的
会議での議論の流れや決定事項、そしてそれに至った理由をツリー構造で可視化する。

### 実装方法
1. **決定ポイントの特定**
   - 「決定しました」「採用します」「〜することになりました」などの表現を検索
   - 「AかBか」「どちらにするか」などの選択肢を特定

2. **理由・根拠の抽出**
   - 決定前後の文脈から「なぜなら」「理由は」「〜だから」を探す
   - メリット・デメリットの議論を収集

3. **代替案の特定**
   - 「他には」「別の方法として」「もう一つの選択肢」などを探す

### 出力形式(Markdown)
```markdown
### 決定事項: [決定内容]
**決定日時**: [タイムスタンプ]
**決定者/合意者**: [人名リスト]

#### 選択した選択肢
- **選択**: [選択A]
- **理由**: 
  - [理由1]
  - [理由2]

#### 検討された代替案
1. **選択肢B**
   - メリット: [...]
   - デメリット: [...]
   - 不採用理由: [...]

2. **選択肢C**
   - メリット: [...]
   - デメリット: [...]
   - 不採用理由: [...]

#### 次のアクション
- [アクション1]
- [アクション2]
```

## 感情分析

### 目的
発言者の感情（ポジティブ、ネガティブ、ニュートラル）を分析し、特定のトピックに対する参加者の隠れた感情を把握する。

### 実装方法
1. **感情指標語の検出**
   - ポジティブ: 「素晴らしい」「良い」「期待」「嬉しい」「賛成」「可能性」
   - ネガティブ: 「難しい」「問題」「課題」「懸念」「心配」「リスク」
   - 強調: 「非常に」「とても」「すごく」「本当に」

2. **話者別感情スコアの計算**
   ```python
   score = (positive_count - negative_count) / total_statements
   ```

3. **トピック別感情の集計**
   - 特定のキーワード(製品名、提案など)周辺の感情を分析

### 出力形式
```markdown
### 感情分析サマリー

#### 全体的な雰囲気
- **ポジティブ度**: 65% (ポジティブな発言が多い)
- **議論の熱度**: 高 (強調表現が頻出)

#### 話者別感情スコア
| 話者 | ポジティブ | ニュートラル | ネガティブ | 総合スコア |
|------|-----------|-------------|-----------|-----------|
| 田中さん | 45% | 35% | 20% | +0.25 |
| 佐藤さん | 30% | 50% | 20% | +0.10 |

#### トピック別感情
**新製品提案**
- 感情傾向: やや懸念あり
- 主な懸念: 「開発期間が短い」「リソース不足」
- 主な期待: 「市場での需要は高い」「差別化できる」
```

## キーワード・トピックトレンド分析

### 目的
頻出するキーワードやトピックを抽出し、組織内で現在注目されているテーマを特定する。

### 実装方法
1. **形態素解析**
   - 名詞、動詞、形容詞を抽出
   - ストップワード(「です」「ます」など)を除去

2. **重要度計算**
   - TF-IDF または単純頻度
   - 共起語分析(一緒に出現する語)

3. **トピッククラスタリング**
   - 関連するキーワードをグループ化
   - 主要トピックを3-5個抽出

### 出力形式
```markdown
### キーワード分析

#### 頻出キーワード TOP20
1. 製品開発 (28回)
2. スケジュール (22回)
3. 予算 (18回)
4. リソース (15回)
5. マーケティング (14回)
...

#### 抽出されたトピック

**トピック1: 製品開発戦略**
- 関連キーワード: 製品開発、新機能、ロードマップ、リリース
- 議論の要点: 次期製品の機能とスケジュールについて

**トピック2: リソース配分**
- 関連キーワード: 予算、人員、外注、優先順位
- 議論の要点: 限られたリソースの効率的な配分

**トピック3: マーケット戦略**
- 関連キーワード: マーケティング、競合、ターゲット、価格
- 議論の要点: 市場投入戦略の検討
```

## 貢献度・影響力分析

### 目的
誰が最も多く発言したか、誰の意見が最終的な決定に繋がったかを分析する。

### 実装方法
1. **発言量の測定**
   - 文字数、発言回数をカウント
   - 話者別に集計

2. **影響力の評価**
   - 決定事項への言及率
   - 他の参加者からの参照頻度(「〜さんが言ったように」)
   - 質問への回答率

3. **エンゲージメント指標**
   - 質問数
   - 提案数
   - 合意・反対の表明数

### 出力形式
```markdown
### 貢献度・影響力分析

#### 発言量ランキング
1. 田中部長: 3,450文字 (全体の35%)
2. 佐藤課長: 2,100文字 (全体の21%)
3. 鈴木さん: 1,800文字 (全体の18%)

#### 影響力スコア
| 参加者 | 提案数 | 質問数 | 決定への貢献 | 影響力スコア |
|--------|--------|--------|-------------|-------------|
| 田中部長 | 5 | 12 | 高 | 8.5/10 |
| 佐藤課長 | 3 | 8 | 中 | 6.2/10 |
| 鈴木さん | 7 | 5 | 高 | 7.8/10 |

#### キーパーソン分析
- **最多発言者**: 田中部長 (リーダーシップを発揮)
- **最多提案者**: 鈴木さん (7件の具体案を提示)
- **意思決定への影響**: 田中部長、鈴木さんの提案が最終決定に反映
```

## SWOT分析

### 目的
議事録の内容を「強み」「弱み」「機会」「脅威」の4つに分類して分析する。

### 実装方法
1. **カテゴリ別キーワードマッピング**
   - 強み: 「優位性」「強み」「できる」「成功」「実績」
   - 弱み: 「課題」「問題」「不足」「弱い」「改善」
   - 機会: 「チャンス」「市場」「需要」「成長」「可能性」
   - 脅威: 「競合」「リスク」「脅威」「減少」「規制」

2. **文脈解析**
   - キーワード周辺の文章を抽出
   - ポジティブ/ネガティブの判定

### 出力形式
```markdown
### SWOT分析

#### 強み (Strengths)
1. **技術力の高さ**
   - 「当社の開発チームは業界トップレベルの技術を持っている」
   - 根拠: 過去3年の製品品質評価

2. **顧客基盤の厚み**
   - 「リピート率85%と高い顧客ロイヤリティ」

#### 弱み (Weaknesses)
1. **マーケティング体制の不足**
   - 「営業人員が足りず、新規開拓が進んでいない」
   - 影響: 市場シェア拡大の遅れ

2. **開発リソースの制約**
   - 「エンジニアの稼働率が限界に達している」

#### 機会 (Opportunities)
1. **新市場の拡大**
   - 「アジア市場での需要が急増中」
   - 潜在規模: 年間50億円市場

2. **規制緩和**
   - 「新法施行により参入障壁が低下」

#### 脅威 (Threats)
1. **競合の台頭**
   - 「X社が類似製品を半額で投入予定」
   - 対策検討中

2. **原材料価格の高騰**
   - 「主要部材が20%値上げ」
```

## ロジックツリー分析

### 目的
問題を構成要素に分解し、その原因や解決策を樹形図で整理する。

### 実装方法
1. **メイン問題の特定**
   - 「問題は」「課題は」などの表現から主要課題を抽出

2. **原因の階層化**
   - 「なぜ」「原因は」の連鎖を追跡
   - Why-Why分析(5回のなぜ)

3. **解決策の構造化**
   - 各原因に対する対策を紐付け

### 出力形式
```markdown
### ロジックツリー分析

#### 問題: 売上が目標を20%下回っている

##### レベル1: 主要原因
1. **新規顧客獲得数の減少**
   ├─ レベル2: 中間原因
   │  ├─ 広告予算の削減
   │  │  └─ レベル3: 根本原因
   │  │     ├─ 全社的なコスト削減方針
   │  │     └─ 解決策: 費用対効果の高い施策への集中
   │  └─ 営業人員の不足
   │     └─ 解決策: 即戦力の中途採用

2. **既存顧客の離反率上昇**
   ├─ レベル2: 中間原因
   │  ├─ サポート品質の低下
   │  │  └─ 解決策: CS部門の増員とトレーニング
   │  └─ 競合製品への流出
   │     └─ 解決策: 製品機能の差別化

#### 優先アクション
1. [高優先] CS部門の即時増員 (期待効果: 離反率10%削減)
2. [中優先] 費用対効果の高い施策への予算再配分
3. [低優先] 長期的な製品改善計画の策定
```

## プロセスマイニング

### 目的
議事録から「誰が」「いつ」「何をしたか」を抽出し、業務プロセスを可視化・分析する。

### 実装方法
1. **アクション・イベントの抽出**
   - 動詞+目的語のパターン検出
   - 完了形(「〜しました」)と予定形(「〜します」)を区別

2. **タイムライン構築**
   - タイムスタンプまたは順序で並べ替え
   - 並行処理と直列処理を識別

3. **ボトルネック分析**
   - 長時間かかったステップ
   - 繰り返し議論されたトピック

### 出力形式
```markdown
### プロセスマイニング分析

#### 検出されたプロセスフロー

```text
[開始] 
  ↓
[課題の共有] (田中部長, 5分)
  ↓
[原因分析] (全員, 15分) ← ボトルネック
  ├─ [データ確認] (佐藤課長)
  └─ [現場ヒアリング] (鈴木さん)
  ↓
[解決策の提案] (全員, 10分)
  ├─ 案A: 短期対策
  └─ 案B: 長期対策
  ↓
[意思決定] (田中部長, 3分)
  ↓
[アクション割り当て] (5分)
  ├─ 田中部長: 予算確保
  ├─ 佐藤課長: 実行計画作成
  └─ 鈴木さん: チーム編成
  ↓
[終了]
```

#### ボトルネック分析
1. **原因分析フェーズ (15分)**
   - 理由: データ不足により議論が停滞
   - 改善策: 事前の資料準備を徹底

2. **繰り返し議論: 予算配分**
   - 3回議論が戻った
   - 原因: 各部門の優先順位が不一致
   - 改善策: 事前に部門間調整を実施

#### プロセス効率化の提案
- 会議前: 必要データの事前共有
- 会議中: タイムボックス設定(各議題15分以内)
- 会議後: 決定事項の即日共有
```

## テキストマイニング

### 目的
自然言語処理技術を用いて、大量のテキストから有益な情報を抽出する。

### 実装方法
1. **共起ネットワーク分析**
   - 一緒に出現する単語のペアを特定
   - ネットワーク図で関連性を可視化

2. **センテンス重要度ランキング**
   - 複数のキーワードを含む文
   - 決定事項や重要な発言

3. **時系列でのトピック変化**
   - 会議の前半・中盤・後半でのトピック推移

### 出力形式
```markdown
### テキストマイニング分析

#### 共起ネットワーク(主要な語の関連性)
- **「製品」** ←→ 「開発」(15回)、「品質」(12回)、「価格」(8回)
- **「顧客」** ←→ 「満足度」(10回)、「フィードバック」(7回)
- **「スケジュール」** ←→ 「遅延」(6回)、「リスク」(5回)

#### 重要センテンス TOP10
1. [32:15] 「今期の最重要課題は顧客満足度の向上であり、すべての施策はこれを軸に判断する」(田中部長)
2. [45:30] 「競合のX社が来月新製品を投入するため、我々も開発を前倒しする必要がある」(佐藤課長)
3. [56:10] 「予算は当初計画の80%に削減されたが、優先順位を明確にすれば対応可能」(鈴木さん)

#### トピック推移分析
| 時間帯 | 主要トピック | 特徴 |
|--------|-------------|------|
| 前半(0-20分) | 状況共有 | 事実ベースの報告が中心 |
| 中盤(20-40分) | 課題分析 | 問題の深掘りと原因究明 |
| 後半(40-60分) | 解決策・意思決定 | 具体的なアクションの決定 |

#### 感情・トーンの変化
```text
[感情スコア推移]
ポジティブ  ▓▓░░░░░░▓▓▓▓▓
ニュートラル ░░▓▓▓▓▓▓░░░░░
ネガティブ  ░░░░░▓▓░░░░░░
            前半  中盤  後半
```
- 前半: ニュートラル(事実共有)
- 中盤: やや ネガティブ(課題認識)
- 後半: ポジティブ(解決策への合意)
```

## フレームワーク選択ガイド

会議の種類に応じて、以下のフレームワークを優先的に使用してください:

| 会議タイプ | 推奨フレームワーク(優先順) |
|-----------|--------------------------|
| **戦略会議** | 1. SWOT分析 2. 意思決定ツリー 3. キーワード分析 |
| **ブレインストーミング** | 1. テキストマイニング 2. キーワード分析 3. 感情分析 |
| **進捗報告** | 1. プロセスマイニング 2. 貢献度分析 3. ロジックツリー |
| **問題解決** | 1. ロジックツリー 2. SWOT分析 3. 意思決定ツリー |
| **クライアント打ち合わせ** | 1. 感情分析 2. 意思決定ツリー 3. アクション抽出 |
| **採用面接** | 1. 感情分析 2. キーワード分析 3. (通常の分析は不要) |
| **レビュー会議** | 1. プロセスマイニング 2. 貢献度分析 3. SWOT分析 |
| **製品議論** | 1. 意思決定ツリー 2. SWOT分析 3. ロジックツリー |
| **一般的な会議** | 1. 基本分析のみ 2. キーワード分析 3. アクション抽出 |

各分析は会議の内容と目的に応じて柔軟に適用してください。
